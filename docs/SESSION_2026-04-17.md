### F3 — Machine-readable surfaces (JSON API + Raw MD + MCP)
**What:** Three doors into the same pack content.
- JSON API: `GET /api/packs?type=&pattern=&trust=&q=&limit=&offset=`, `GET /api/packs/<slug>` (canonical `{pack: Pack}` envelope)
- Raw markdown: `GET /packs/<slug>/raw` with 19 stable H2 anchors for section-level retrieval
- MCP server: `attrition-mcp` with 5 tools — `resolve_pack_id`, `get_pack`, `get_pack_section`, `search_packs`, `compare_packs`
**Why:** Agents are first-class readers. Give them their own retrieval surface, not a browser scrape.
**Shipped:** `src/lib/pack-markdown.ts` serializer; 3 route handlers; `mcp-server/` npm package (9 files); AbortController timeouts; 1MB/5MB byte caps; structured `{error: {code, message}}`; zod validation.
**Status:** ✅ — 8/8 MCP adversarial scenarios pass; MCP package ready, not yet on npm.
**Hooks:** *One pack, many doors.* · *Agents need address books too.*

### F4 — Installation & distribution (CLI + AGENTS.md)
**What:** `attrition` CLI with subcommands `pack install | list | search | verify`. Writes `.claude/skills/<slug>/SKILL.md`, appends idempotent marker-delimited fragment to `AGENTS.md`, pins versions in `.attrition/installed.json`.
**Why:** Copy-paste is not a commitment. Install is — atomic, idempotent, reversible.
**Shipped:** `cli/` npm package (13 files); atomic write primitive (`tmp + rename` with Windows EPERM retry-with-backoff); idempotent marker upsert with corruption detection; slug-traversal guard at every entry.
**Status:** ✅ — 18/18 scenario tests including concurrency + adversarial.
**Hook:** *Install atomic. Install idempotent.*

### F5 — Trust & verification (eval gate)
**What:** Automated gate between Community and Verified tiers. Goldens table + 3 seeds (harness, universal shape, eval pack). 7-variant assertion discriminated union (substring-present/absent, regex-match, field-equals, field-nonempty, injection-probe, llm-judge-rubric). Promotion rule: `runs ≥ 10 ∧ passRate ≥ 0.95 ∧ injectionSurface ≠ "high" ∧ no blocking failures`.
**Why:** Trust must be measured by a third party, not declared by the subject.
**Shipped:** `convex/{goldens.ts, eval.ts, packTrust.ts}` + `src/lib/eval-assertions.ts` (pure engine, unit-testable without Convex runtime); backward-compat schema extensions only. `golden-eval-harness` seed pack demoted `Verified → Community` to enforce the rule at seed time.
**Status:** ✅ — 5/5 verify scenarios (happy / sad / adversarial / low-N / high-surface).
**Hook:** *Claims aren't proof. Measurements are.*

### F6 — Knowledge pedagogy (4-line Why + glossary)

### F7 — Failure-mode catalog (production scar tissue)
**What:** Every change row and every pack carries its failure-mode table: **Symptom** (what you see at 3am) · **Trigger** (what caused it) · **Prevention check** (what senior devs add proactively) · **Tier** (Jr / Mid / Sr / Staff — who learns this by pager at what career stage). Cross-cutting 8-point CLAUDE.md reliability checklist (BOUND / HONEST_STATUS / HONEST_SCORES / TIMEOUT / SSRF / BOUND_READ / ERROR_BOUNDARY / DETERMINISTIC) mapped to agent-specific failure classes (unbounded Maps OOM, fake 201s become false beliefs, inflated scores cause skipped verification, etc.).
**Why:** A pattern catalog without failure modes is a textbook. With failure modes, it's a production playbook. Compresses years of on-call scar tissue into browsable knowledge so juniors don't re-learn by incident.
**Shipped:** 15 failure-mode rows in `ct_2026-04-17.md` + 8-point cross-cutting table; `FailureMode` type added to `ChangeRow` in `CHANGE_TRACE.md §3`.
**Status:** ✅ on change-trace side; 🟡 `Pack.failureModes` still `string[]`, pending upgrade to structured shape.
**Hook:** *"I'd think about X because in production Y can happen."* (the sentence that moves you from mid to senior)

### F8 — Change-trace catalog (Pillar 2 — LightRAG for your own sessions)
**What:** Second catalog pillar. 4-column rows (Scenario / Files / Changes / Why) capturing what a coding session changed. Ctrl+F by symbol / file / pack / hook. Cross-references into Pillar 1 via `packsReferenced[]`. Same infra reused (Convex tables, MCP tools, CLI, raw MD, `/traces` browser route).
**Why:** Vibe-coding with agents produces drift. You forget what got renamed when. The catalog makes your own history ctrl+F-able. Bridges setup (cheap) → understanding (expensive) → measured control (hard).
**Shipped (design):** `docs/CHANGE_TRACE.md` — thesis, schema with `WhyExplanation` + `FailureMode`, 6 ASCII mock views (directory, detail, symbol filter, file filter, raw MD, MCP responses), capture workflows (manual → extractor → PostToolUse hook), quality bar.
**Shipped (seed):** `docs/traces/ct_2026-04-17.md` — this session rendered as the first real trace: 15 rows, 38 files, 27 symbols, 4 packs referenced.
**Status:** ✅ design + one real seed; 🟡 M7 code (Convex tables + `attrition trace log` CLI + `/traces` route + MCP tools) not yet built.
**Hook:** *Same catalog, new content.*

### F9 — Storage ADR (relational, never graph)
**What:** ADR-001 captured. Primary store: **relational** (Convex tables / Postgres / SQLite FTS5). Secondary: pgvector / sqlite-vec sidecar *if* measured demand exists. Explicitly rejected: Neo4j / LightRAG primary, JSON-only agent retrieval, pure vector DB.
**Why:** 95% of queries on a 4-column table are filter + FTS. Graph DBs add 10× ops cost for 2–3 hop queries that are SQL joins, and push teams toward LLM-only entry which regresses browsability.
**Shipped:** `docs/CHANGE_TRACE.md §5`.
**Status:** ✅ accepted.
**Hook:** *A 4-column table is a table.*
