## Full 4-line Why per row

### Row 1 — Suspense-hid-everything

| | |
|---|---|
| Plain | Put the chip row on the server so the page shows up before JavaScript runs. |
| Analogy | Like painting your storefront window during the night — the sign has to be ready when customers arrive, not while they wait outside. |
| Principle | `useSearchParams` in a client component forces the whole subtree to client-only render. If the content does not need interactivity, it belongs on the server. |
| **Hook** | **Verify what the user sees, not what the server sent.** |

### Row 2 — Git binding missing

| | |
|---|---|
| Plain | A project started from the command line is not watching your Git repo automatically. |
| Analogy | Like installing a mailbox without telling the post office your address — letters go nowhere. |
| Principle | `vercel --prod` creates a Vercel project without a Git binding. The GitHub App installation is not the same as per-project repo linkage; both must be in place. |
| **Hook** | **CLI-created projects don't auto-bind Git.** |

### Row 3 — Root tsconfig swept subpackages

| | |
|---|---|
| Plain | The root type-check was grading files that live in a separate package with different dependencies. |
| Analogy | Like grading a student's essay against every textbook in the library instead of the one the assignment used. |
| Principle | When a repo contains standalone TypeScript packages (each with its own `tsconfig` + `node_modules`), the root `tsconfig.include` glob must explicitly exclude their paths. `**/*.ts` is too greedy. |
| **Hook** | **Exclude standalone packages from root tsconfig.** |

### Row 4 — Classic webhook red herring

| | |
|---|---|
| Plain | I checked the wrong place. GitHub Apps don't show up in the classic-webhook list. |
| Analogy | Like checking the paper mailbox for an email. Email never arrives there. |
| Principle | GitHub App event subscriptions and classic repo webhooks are two different integration mechanisms. `GET /repos/*/hooks` only lists the latter. Verify App integrations by pushing a commit and watching the actual deploy target, not by querying `/hooks`. |
| **Hook** | **GitHub Apps don't show up in /hooks.** |

## Cross-cutting checks that fired

From CLAUDE.md's 8-point reliability list, these were actively engaged:

- **HONEST_STATUS** — the SSR rewrite doesn't synthesize counts; everything in the chip row is measured from `getAllPacks()`, `getAllTraces()`, and the publisher set.
- **DETERMINISTIC** — URL params sorted/serialized via stdlib `URLSearchParams` for chip hrefs so the same filter state always produces the same href.
- **ERROR_BOUNDARY** — `<Suspense>` intentionally removed; `buildDirectoryData()` either returns or throws; no silent empty state.
