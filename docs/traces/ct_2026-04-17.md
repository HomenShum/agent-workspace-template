## Glossary (for first-time readers — plain English + industry analog)

| Term | What it means here | Relatable to |
|---|---|---|
| **Pack** | One installable entry in the catalog. Contains docs, an install command, optional measured metrics, and a recommended pattern for a single production problem. | Claude Code skill · Convex component · Cursor rule · shadcn component · npm package for a pattern |
| **Harness** *(agent-specific)* | Everything around a **language model** that makes it a working **agent**: system prompt, tool definitions, orchestration loop, memory / context management, verification, guardrails. Only applies to packs where `packType: "harness"`. A UI pack or RAG pack does **not** have a harness. | The "operating system" around the "CPU" (where the LLM is the CPU). Anthropic's Claude Agent SDK calls itself a "general-purpose agent harness." LangChain DeepAgents uses the term the same way. Term entered mainstream use in early 2026. |
| **Pack types** | Catalog entries come in 8 flavors: **harness** (agent orchestrations), **ui** (interface patterns), **rag** (retrieval patterns), **eval** (evaluation patterns), **reference** (decision trees / guides), **data**, **design**, **security**. Only `harness` packs carry the `contract` + `layers` fields. | shadcn components (`ui`) · LangChain chains (`harness`) · LlamaIndex retrievers (`rag`) · OpenAI Evals (`eval`) |
| **Trust tier** | `Verified` (passed our automated eval gate with ≥95% pass rate over ≥10 runs) or `Community` (self-published, unverified). | npm "verified publisher" · Docker Official Images |
| **Skill** | Claude Code's filesystem convention: `.claude/skills/<name>/SKILL.md`. Claude Code picks it up automatically. Our `attrition pack install` writes packs into this format. | Same as Claude Code skills (60k+ repos already use the `AGENTS.md` + `.claude/skills/` pattern) |
| **AGENTS.md** | Repo-root file that CC / Cursor / Codex read for project conventions. Our CLI appends marker-delimited fragments, one per installed pack. | Same as the AGENTS.md convention across the agent-coding ecosystem. |
| **Trace** | One coding session's changes captured as rows (Scenario / Files / Changes / Why). *This document is a trace.* | Git log + commit messages, but human-readable, cross-referenced, and browseable by ctrl+F. |
| **Contract** | Required outputs, token budget, permissions, completion conditions, output path — for a pack that runs a harness. Turns fuzzy LLM calls into bounded function calls. | Function signature · API contract · Tongyi NLA execution contract |
| **Canonical pattern** | One of the 5 Anthropic-named agent patterns: prompt-chaining, routing, parallelization, orchestrator-workers, evaluator-optimizer. | Anthropic "Building effective agents" taxonomy |
| **Registry / Catalog** | The website + MCP + API at `agentworkspace.attrition.sh` that hosts packs. | Convex Components directory · Context7 · npm registry |
| **Hook (in a Why row)** | ≤6-word memorable imperative — the ctrl+F anchor for the lesson. | Like a slogan or mnemonic. "Measure twice, cut once." |
| **Failure mode** | What can go wrong in production that this change prevents. Every mature check in the catalog exists because someone got paged at 3am. | Post-mortem finding · production runbook · "things we learned the hard way" list |
| **Maturity tier** | Who typically learns a given check by pain: **Jr** (happy path only, not yet paged) · **Mid** (paged once, now thinks "what if X is down?") · **Sr** (seen 5–10 failure classes, adds checks proactively) · **Staff** (seen failures at scale, writes checklists so others don't re-learn). | Standard engineering ladder framing — Junior / Mid / Senior / Staff |

## Failure modes — accumulated scar tissue per row

> *"Every checklist item above traces back to a specific production failure someone got paged for at 3am."*
>
> **What this section does:** for every change row, name the production failure that forces that check to exist. A junior who consults the catalog before shipping gains the same instinct as a senior who got paged — without the page. That is the catalog's biggest leverage: **compress years of on-call scar tissue into browsable, installable knowledge.**
>
> **How engineers actually learn this:**
> - **Jr** — writes code that works in dev. Happy path only.
> - **Mid** — been paged once. Thinks "what if X is down?"
> - **Sr** — seen 5–10 failure classes. Adds checks before shipping because they've felt the pain.
> - **Staff** — seen failures at scale. Writes checklists like this one so juniors don't have to learn by incident.
>
> **For interviews:** you don't need to have personally experienced all of these. The distinguishing sentence is *"I'd think about X because in production Y can happen."* Naming the failure mode is what moves you from mid to senior.

### Additional cross-cutting checks (apply to every row)

These are the CLAUDE.md 8-point agentic-reliability checks. Each maps to a production failure class that amplifies in agent loops:

| Check | Failure class it prevents |
|---|---|
| **BOUND** — every in-memory collection has `MAX` + eviction | Unbounded Maps OOM in minutes under agent loops |
| **HONEST_STATUS** — no 2xx on failure paths | Fake 201s become false beliefs in reasoning chains; the agent "thinks" the work succeeded |
| **HONEST_SCORES** — no hardcoded score floors | Inflated scores cause agents to skip verification steps; cascading confidence errors |
| **TIMEOUT** — AbortController + budget gate on every fetch | Agent chain hangs indefinitely on one slow dependency; upstream timeouts amplify |
| **SSRF** — URL validation before fetch | Cloud metadata exfiltration (`169.254.169.254/latest/meta-data/`) via unvalidated URL in a pack source |
| **BOUND_READ** — response size cap on external bodies | Oversized response body buffers into OOM; pack download becomes a DoS vector |
| **ERROR_BOUNDARY** — async error handling on all routes | Unhandled promise rejection crashes the Next.js worker; one bad pack takes the site down |
| **DETERMINISTIC** — sorted-key hashing for content-addressed state | Non-deterministic key order breaks content-addressed storage; cache misses cascade |

**Rule:** any row in this trace that modifies a server boundary, an install path, or a fetch call must tick the relevant subset of these 8 before shipping.

## Full 4-line Why per row

Each Why row obeys the pedagogy schema from `docs/CHANGE_TRACE.md §4` (Plain ≤15 words, Analogy ≤20, Principle ≤20, Hook ≤6).

### Row 1 — Canonical schema pinned

| | |
|---|---|
| Plain | Pick one shape for the data before five workers start using it. |
| Analogy | Like everyone agreeing on paper size before the printers start running. |
| Principle | A shared schema is the contract that lets parallel work converge. Without it every worker drifts. |
| **Hook** | **Pin the shape before the work.** |

### Row 2 — Raw MD + API (M2)

| | |
|---|---|
| Plain | Give the same pack many doors so different tools can read it. |
| Analogy | A book in the library, on Kindle, and as audio — same content, different doors. |
| Principle | Same source of truth, many surfaces. UI and agents are both valid readers. |
| **Hook** | **One pack, many doors.** |

### Row 3 — MCP server (M3)

| | |
|---|---|
| Plain | Let coding agents call the catalog directly, not through a browser. |
| Analogy | Like giving your courier an address book instead of asking them to guess each street. |
| Principle | Agents are first-class readers. Give them their own retrieval surface, not a browser scrape. |
| **Hook** | **Agents need address books too.** |

### Row 4 — CLI (M4)

| | |
|---|---|
| Plain | Turn "copy-paste these files" into one command you can re-run or undo. |
| Analogy | Like a package installer versus emailing everyone a zip and hoping. |
| Principle | Install is a commitment. Make it atomic, idempotent, and reversible at the filesystem boundary. |
| **Hook** | **Install atomic. Install idempotent.** |

### Row 5 — Breadth seed (M5)

| | |
|---|---|
| Plain | Ship packs of different kinds so the catalog stops looking like a toy. |
| Analogy | A bookstore with only fiction — nobody believes it's a real store until biology sits on the next shelf. |
| Principle | The shape is proved by breadth, not depth. One of each type beats five of one. |
| **Hook** | **Prove breadth before depth.** |

### Row 6 — Eval gate built (M6)

| | |
